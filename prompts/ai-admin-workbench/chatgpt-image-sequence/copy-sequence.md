# AI 行政工作台 ChatGPT 生圖貼上順序

使用方式：

正式 CDP 批次生成以 `D:\projects\browser-automation-workflow\scratch\ai-admin-workbench-final-15\prompts` 為準；該目錄目前為 16 張，且第 15 張為附錄、第 16 張為結語。本檔保留作人工貼上參考。

3. 每張圖產生後下載，依序命名為 `01-cover.png`、`02-why-workbench.png` 等。

## 00 Style Primer

```text
接下來我要製作一組 16:9 橫式簡報投影片圖片，主題是「AI 行政工作台」，對象是台灣國小校內行政團隊與採購審查者。

請從現在開始，所有圖片都維持同一套視覺風格：

- 溫暖、清楚、可信、務實、有制度感的國小行政提案簡報風格
- 背景以米白紙感、淺暖灰、深綠、墨黑、柔金色、少量行政藍為主
- 使用乾淨的資訊圖表、資料夾結構、行事曆、檢核表、公文資料夾、Codex 桌面版工作區、瀏覽器工作台、檔案樹與對話協作面板意象
- 視覺要像校內行政簡報，不要像企業科技廣告
- 避免科幻感、霓虹感、過多程式碼、密密麻麻的小字
- 如果有文字，請使用繁體中文，字要大、清楚、少量
- 不要加入 logo、水印、品牌標誌、真實個資、帳號、token、日曆 ID、公文號
- 每張圖都要是完整 16:9 投影片頁面，可以直接作為簡報圖片使用
- 以下規則是整份簡報的通用規則，不是單張投影片特例
- 每張圖的主體必須是行政邏輯：案例證據、流程、資料、安全、採購或推廣計畫
- 若本次已上傳頂溪國小四精靈參考圖，請只把它當作四個角色的造型參考，不要產生校徽、logo、印章或任何額外品牌標誌
- 每張圖右上角保留小型校徽後製空間，左下角保留頁碼後製空間；這兩個角落不要放文字、圖示、數據、精靈或重要物件
- 四精靈為品牌識別，解剖結構不可改：不可新增或刪除手、腳、眼睛、翅膀、尾巴、角、耳朵、鰭、爪或任何額外肢體
- 品牌識別絕對不能反客為主；每張圖的主體必須是行政案例、流程、資料、表格或提案理由
- 精靈可選擇 0-4 隻小型點綴；只有在不影響文字、資料、流程與主訊息時才加入，資訊密集頁可完全不放
- 精靈只能當小型輔助點綴，單隻高度約不超過投影片高度 10%，四隻合計約不超過畫面 15%
- 精靈不要放在畫面中央當主角，優先放在邊角、桌面前景或小型提示位置
- 避免可愛海報感、巨大精靈臉、貼紙堆疊、漫畫式驚嘆符、過度飽和色彩
- 光精靈：黃色圓身、一條發光觸角、兩手、兩腳、一對黃色翅膀、沒有尾巴
- 火精靈：橘紅火焰身、兩手、兩腳、兩顆正面牙齒、沒有翅膀、沒有尾巴、不可長出額外火焰肢體
- 海精靈：藍色水滴身、單眼、兩手、兩腳、沒有翅膀、沒有尾巴、不可多眼
- 樹精靈：橘色樹幹身與綠色樹冠、兩手、兩腳；葉子、鳥、果實只是裝飾，不可變成額外手腳
- 若動作很難畫，請簡化姿勢或縮小精靈，不可為了拿道具而增加手、尾巴或支撐肢體
- 所有落地應用或實作成果頁都必須在主標題清楚標示「案例」；標題需以「案例：」或「場租案例一／二／三：」開頭，不可省略
- 每個案例頁都要明確呈現 AI 工作台價值，讓觀眾看得出 AI 如何讀取、整理、檢核、草擬、串接 API、產出結果或支援人工確認
- 若案例使用 API，畫面中要呈現 API 在流程中的位置，不可只畫出最終介面
- 使用「模擬執行」；若必須出現英文，寫成「模擬執行（dry-run）：先預覽結果，不直接改原始資料」
- 縮寫第一次出現必須解釋，例如「場租管理系統（VRMS）」與「Venue Rental Management System」

請先確認你會在後續每張圖中延續這個風格。不要先產生圖片。
```

## Appendix Cases

```text
請產生一張 16:9 橫式投影片圖。

投影片文字：
標題：「附錄：補充案例」

版面：
- 採 2x2 四格卡片版型
- 每格只放四列：案例、情境、用途、亮點
- 文字要大、清楚、可讀，不要塞太多字
- 品牌識別只能是很小的角落點綴，可以不放精靈，不可搶走四格案例內容

四個案例：

1. 案例：財產盤點與消耗用品管理
情境：總務處
用途：盤點準則、通知、勾核工具與領用紀錄
亮點：從清冊到回收表，流程可查可交接

2. 案例：家長會選舉系統
情境：校務治理
用途：線上投票、匿名防重複、即時統計
亮點：安全、可稽核、可匯出結果

3. 案例：行政簡報製作工作流
情境：提案與成果報告
用途：brief、頁面大綱、圖片提示詞、PDF 輸出
亮點：不是一鍵亂生，而是可審稿、可追溯

4. 案例：每日工作檢核與資料夾整理
情境：文書組例行維護
用途：每日檢核表、資料夾盤點、年度交接
亮點：把例行工作變成可追蹤制度

限制：
- 只使用上列四個補充案例，不要自行加入其他產品、舊工作流、個人專案、旅遊、遊戲或無關案例
```
