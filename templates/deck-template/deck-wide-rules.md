# Deck-Wide Rules Template

Copy this into a prompt pack README or style primer before writing individual slide prompts.

```text
- The deck is a formal presentation for <audience>. Keep it readable, credible, and aligned with the brief.
- The main subject of each slide must be the slide message, not decoration or brand assets.
- Brand identity must support the message only. It must never become the composition, theme, or visual focus.
- Reserve overlay-safe corners: keep the top-right corner clear for the official crest or logo, and keep the bottom-left corner clear for the page-number badge. Do not place text, icons, metrics, mascots, charts, or important objects in those two corners.
- Mascots or characters are optional. Use them only when they do not interfere with text, data, workflow, or the slide message.
- Use restrained colors, clear spacing, and information-design layout appropriate for <domain>.
- Use audience-friendly language. Avoid unexplained technical terms.
- Explain acronyms on first use.
- Every case or workflow example slide must make the workflow value explicit.
- If a case uses an API or external system, show the data exchange or API step rather than only showing the final interface.
```

Per-slide footer reminder:

```text
Do not create a page number badge. Do not create any crest, logo, seal, stamp, or official emblem. Leave the bottom-left and top-right corners empty for code overlay.
```
