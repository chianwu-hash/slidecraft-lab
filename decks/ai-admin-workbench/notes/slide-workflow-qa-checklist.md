# AI 生圖簡報 QA Checklist

用途：每次用 ChatGPT 網頁產生全頁簡報圖片後，先用本表檢查，再決定是否重生單頁或輸出正式 PDF。

## A. 整份簡報總覽

- [ ] 張數正確，頁序正確。
- [ ] 每張圖都是 16:9 橫式。
- [ ] 整體色調一致，沒有某幾張突然變成廣告海報或卡通風。
- [ ] 主題線清楚，觀眾能看出這是一份正式行政提案。
- [ ] 附錄與結語位置正確。

## B. 後製識別

- [ ] 右上角校徽大小一致。
- [ ] 右上角校徽沒有變形、變色或被其他物件干擾。
- [ ] 左下角頁碼大小一致。
- [ ] 左下角頁碼沒有遮住數據、流程、標題或重要說明。
- [ ] 圖中沒有 ChatGPT 自行生成的假校徽、假印章或假 logo。

## C. 文字與內容

- [ ] 主要文字為繁體中文。
- [ ] 標題可讀，不被頁碼、校徽、精靈或圖表遮擋。
- [ ] 每張案例頁標題清楚標示「案例」或「場租案例」。
- [ ] 工程術語已白話化，例如使用「模擬執行」而不是只寫 `dry-run`。
- [ ] 縮寫第一次出現有說明，例如 `場租管理系統（VRMS）`。
- [ ] 沒有校長行事曆、個資、帳號、token、公文號或敏感資料。

## D. AI 工作台價值

- [ ] 每張案例頁都看得出 AI 做了什麼。
- [ ] 若使用 API，流程中有畫出 API 或資料交換，不只呈現結果畫面。
- [ ] 有保留人工確認、承辦人判斷或安全檢核。
- [ ] 沒有把 AI 描述成完全自動取代行政人員。

## E. 品牌精靈

- [ ] 精靈可有可無，沒有為了放精靈而犧牲內容。
- [ ] 精靈沒有位於畫面中央並搶走重點。
- [ ] 精靈沒有遮住文字、表格、流程或數據。
- [ ] 光精靈保留黃色身體、一條發光觸角、一對翅膀、兩手兩腳。
- [ ] 火精靈沒有翅膀、尾巴或額外火焰肢體。
- [ ] 海精靈維持單眼、兩手兩腳。
- [ ] 樹精靈維持兩手兩腳，葉片、鳥與果實不被畫成額外肢體。

## F. 重生判斷

符合以下任一條件，建議重生該頁：

- [ ] 重要文字錯字或語意錯誤。
- [ ] 頁碼或校徽預留區被重要物件佔用。
- [ ] 精靈變形、長出多餘手腳、翅膀或尾巴。
- [ ] 品牌識別反客為主。
- [ ] 案例內容與處室、用途或流程不符。
- [ ] 觀眾可能看不出這張與 AI 行政工作台有關。
