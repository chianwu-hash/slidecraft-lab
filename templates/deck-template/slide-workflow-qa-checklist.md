# Slide Workflow QA Checklist

Use this checklist after generating slide images and again after applying any overlays.

## Whole Deck

- [ ] Slide count is correct.
- [ ] Slide order is correct.
- [ ] All slides use the intended aspect ratio.
- [ ] The deck has a coherent visual tone.
- [ ] The deck still matches the brief and intended audience.

## Fixed Overlays

- [ ] Page numbers are consistent.
- [ ] Page numbers do not cover important content.
- [ ] Logo or crest placement is consistent.
- [ ] Logo or crest is not distorted, recolored, or duplicated by the image model.
- [ ] Prompted overlay-safe zones were actually left clear.

## Text And Content

- [ ] Main slide titles are readable.
- [ ] Required case labels, section labels, or subtitles are present.
- [ ] Audience-facing terms are understandable.
- [ ] Acronyms are explained on first use.
- [ ] No sensitive personal data, credentials, IDs, or private schedules are visible.

## Brand And Mascots

- [ ] Brand elements support the message and do not dominate the slide.
- [ ] Mascots or characters are optional and do not block text, data, or workflows.
- [ ] Mascots or characters preserve required colors, anatomy, and silhouettes.
- [ ] No invented logos, seals, marks, or unrelated brand elements appear.

## Workflow Value

- [ ] Each case slide makes the workflow value explicit.
- [ ] If an API is part of the case, the workflow shows the API or data exchange.
- [ ] Human confirmation, review, or approval remains visible where needed.
- [ ] The slide does not imply unsafe full automation.

## Regenerate This Slide If

- [ ] Important text is wrong or unreadable.
- [ ] Overlay-safe zones are occupied.
- [ ] Brand elements dominate the slide.
- [ ] A mascot or character is malformed.
- [ ] The case is factually wrong.
- [ ] Viewers would not understand why this slide belongs in the deck.
